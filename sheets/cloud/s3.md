# S3 (Object Storage)

AWS Simple Storage Service providing scalable object storage with an HTTP API, supporting versioning, lifecycle management, access control, presigned URLs, multipart uploads, and cross-region replication.

## aws s3 (High-Level CLI)

```bash
# List buckets
aws s3 ls

# List objects in a bucket
aws s3 ls s3://my-bucket/
aws s3 ls s3://my-bucket/prefix/ --recursive
aws s3 ls s3://my-bucket/ --human-readable --summarize

# Copy files
aws s3 cp file.txt s3://my-bucket/path/file.txt
aws s3 cp s3://my-bucket/path/file.txt ./local-file.txt
aws s3 cp s3://source-bucket/key s3://dest-bucket/key

# Copy with metadata
aws s3 cp file.txt s3://my-bucket/ \
  --content-type "text/plain" \
  --metadata '{"project":"alpha"}' \
  --storage-class STANDARD_IA

# Sync directory
aws s3 sync ./local-dir s3://my-bucket/prefix/
aws s3 sync s3://my-bucket/prefix/ ./local-dir/
aws s3 sync s3://my-bucket/ s3://backup-bucket/ --delete

# Sync with filters
aws s3 sync . s3://my-bucket/ \
  --exclude "*" \
  --include "*.log" \
  --include "*.csv"

# Move
aws s3 mv file.txt s3://my-bucket/
aws s3 mv s3://my-bucket/old-key s3://my-bucket/new-key

# Remove
aws s3 rm s3://my-bucket/file.txt
aws s3 rm s3://my-bucket/prefix/ --recursive

# Create bucket
aws s3 mb s3://new-bucket --region us-east-1

# Remove bucket (must be empty)
aws s3 rb s3://empty-bucket
aws s3 rb s3://non-empty-bucket --force      # delete all objects first
```

## aws s3api (Low-Level CLI)

```bash
# Create bucket with configuration
aws s3api create-bucket \
  --bucket my-bucket \
  --region us-west-2 \
  --create-bucket-configuration LocationConstraint=us-west-2

# Head object (metadata without downloading)
aws s3api head-object --bucket my-bucket --key path/file.txt

# Get object with byte range
aws s3api get-object \
  --bucket my-bucket \
  --key large-file.bin \
  --range bytes=0-1048575 \
  output-first-1mb.bin

# Put object with encryption
aws s3api put-object \
  --bucket my-bucket \
  --key secret.txt \
  --body secret.txt \
  --server-side-encryption aws:kms \
  --ssekms-key-id alias/my-key

# Delete objects (batch)
aws s3api delete-objects --bucket my-bucket --delete '{
  "Objects": [
    {"Key": "file1.txt"},
    {"Key": "file2.txt"},
    {"Key": "file3.txt"}
  ]
}'

# List objects with pagination
aws s3api list-objects-v2 \
  --bucket my-bucket \
  --prefix logs/2024/ \
  --max-keys 1000

# Get bucket location
aws s3api get-bucket-location --bucket my-bucket
```

## Presigned URLs

```bash
# Generate presigned download URL (default 1 hour)
aws s3 presign s3://my-bucket/file.txt

# Presigned URL with custom expiry
aws s3 presign s3://my-bucket/file.txt --expires-in 3600   # 1 hour
aws s3 presign s3://my-bucket/file.txt --expires-in 86400  # 24 hours

# Presigned PUT URL (for upload)
aws s3api put-object \
  --bucket my-bucket \
  --key uploads/user-file.txt \
  --generate-presigned-url \
  --expires-in 3600

# Use presigned URL (no AWS credentials needed)
curl -o downloaded.txt "https://my-bucket.s3.amazonaws.com/file.txt?X-Amz-..."
curl -X PUT -T upload.txt "https://my-bucket.s3.amazonaws.com/uploads/file.txt?X-Amz-..."
```

## Versioning

```bash
# Enable versioning
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# List object versions
aws s3api list-object-versions --bucket my-bucket --prefix file.txt

# Get specific version
aws s3api get-object \
  --bucket my-bucket \
  --key file.txt \
  --version-id "abc123" \
  file-v1.txt

# Delete specific version (permanent)
aws s3api delete-object \
  --bucket my-bucket \
  --key file.txt \
  --version-id "abc123"

# Restore deleted object (remove delete marker)
aws s3api delete-object \
  --bucket my-bucket \
  --key file.txt \
  --version-id "delete-marker-id"
```

## Lifecycle Rules

```bash
# Set lifecycle configuration
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration '{
  "Rules": [
    {
      "ID": "move-to-ia-after-30d",
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Transitions": [
        {"Days": 30, "StorageClass": "STANDARD_IA"},
        {"Days": 90, "StorageClass": "GLACIER"},
        {"Days": 365, "StorageClass": "DEEP_ARCHIVE"}
      ]
    },
    {
      "ID": "expire-temp-after-7d",
      "Status": "Enabled",
      "Filter": {"Prefix": "tmp/"},
      "Expiration": {"Days": 7}
    },
    {
      "ID": "cleanup-old-versions",
      "Status": "Enabled",
      "Filter": {"Prefix": ""},
      "NoncurrentVersionExpiration": {"NoncurrentDays": 30},
      "AbortIncompleteMultipartUpload": {"DaysAfterInitiation": 7}
    }
  ]
}'

# Get lifecycle configuration
aws s3api get-bucket-lifecycle-configuration --bucket my-bucket
```

## Multipart Upload

```bash
# Initiate multipart upload
UPLOAD_ID=$(aws s3api create-multipart-upload \
  --bucket my-bucket \
  --key large-file.bin \
  --query 'UploadId' --output text)

# Upload parts (minimum 5 MB per part, except last)
aws s3api upload-part \
  --bucket my-bucket \
  --key large-file.bin \
  --part-number 1 \
  --upload-id "$UPLOAD_ID" \
  --body part1.bin

aws s3api upload-part \
  --bucket my-bucket \
  --key large-file.bin \
  --part-number 2 \
  --upload-id "$UPLOAD_ID" \
  --body part2.bin

# Complete multipart upload
aws s3api complete-multipart-upload \
  --bucket my-bucket \
  --key large-file.bin \
  --upload-id "$UPLOAD_ID" \
  --multipart-upload '{
    "Parts": [
      {"PartNumber": 1, "ETag": "\"etag1\""},
      {"PartNumber": 2, "ETag": "\"etag2\""}
    ]
  }'

# List in-progress multipart uploads
aws s3api list-multipart-uploads --bucket my-bucket

# Abort multipart upload
aws s3api abort-multipart-upload \
  --bucket my-bucket \
  --key large-file.bin \
  --upload-id "$UPLOAD_ID"

# High-level CLI handles multipart automatically for large files
aws s3 cp large-file.bin s3://my-bucket/ \
  --expected-size 10737418240           # 10 GB hint
```

## Bucket Policy

```bash
# Set bucket policy
aws s3api put-bucket-policy --bucket my-bucket --policy '{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicRead",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/public/*"
    },
    {
      "Sid": "DenyUnencrypted",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "aws:kms"
        }
      }
    }
  ]
}'

# Block all public access
aws s3api put-public-access-block --bucket my-bucket \
  --public-access-block-configuration \
  BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

## Storage Classes

```bash
# Standard              — frequent access, low latency
# Standard-IA           — infrequent access, lower cost, retrieval fee
# One Zone-IA           — single AZ, lower cost than Standard-IA
# Intelligent-Tiering   — auto-moves between tiers based on access
# Glacier Instant        — archive, millisecond retrieval
# Glacier Flexible       — archive, 1-12 hour retrieval
# Glacier Deep Archive   — cheapest, 12-48 hour retrieval

# Set storage class on upload
aws s3 cp file.txt s3://my-bucket/ --storage-class GLACIER

# Restore from Glacier
aws s3api restore-object \
  --bucket my-bucket \
  --key archived-file.txt \
  --restore-request '{"Days": 7, "GlacierJobParameters": {"Tier": "Standard"}}'
```

## Tips

- Enable versioning on critical buckets to protect against accidental deletes and overwrites
- Use lifecycle rules to automatically transition old data to cheaper storage classes and save costs
- Set `AbortIncompleteMultipartUpload` lifecycle rules to clean up abandoned multipart uploads that waste storage
- Block all public access at the account level unless you explicitly need public buckets
- Use presigned URLs for temporary access instead of making objects public or sharing credentials
- For uploads over 100 MB, use multipart upload (the CLI does this automatically with `aws s3 cp`)
- Use `--delete` flag with `aws s3 sync` carefully; it removes files at the destination not present at the source
- Use S3 Intelligent-Tiering for data with unpredictable access patterns to avoid manual lifecycle management
- Enable server-side encryption (SSE-S3 or SSE-KMS) by default using a bucket policy that denies unencrypted puts
- Use `aws s3api head-object` to check metadata and existence without downloading the object
- Prefer `list-objects-v2` over `list-objects` for better performance and pagination support
- Use `--expected-size` hint with large uploads so the CLI can choose optimal part sizes
