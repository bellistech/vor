# G.8032 Ethernet Ring Protection (ERP)

ITU-T standard for sub-50ms Ethernet ring protection switching, providing carrier-grade resilience for metro Ethernet networks using R-APS signaling and RPL blocking.

## Concepts

### Ring Architecture

- **Ring:** Closed loop of Ethernet nodes connected by ring links
- **RPL (Ring Protection Link):** One link in the ring is blocked during normal operation to prevent loops
- **RPL Owner:** Node responsible for blocking the RPL (one end of the RPL)
- **RPL Neighbor:** Node at the other end of the RPL (assists with blocking)
- **Ring Node:** Any other node in the ring
- **Ring Port:** Port connecting a node to the ring (each node has two ring ports: port0 and port1)

### R-APS (Ring APS) Messages

- Carried in CFM (Connectivity Fault Management) frames on a dedicated R-APS VLAN
- Destination MAC: **01:19:A7:00:00:xx** (xx = ring ID)
- EtherType: **0x8902** (CFM)
- OpCode: **40** (R-APS specific)

| R-APS Message | Code | Purpose |
|---------------|------|---------|
| NR (No Request) | 0 | Normal state, no switching action needed |
| NR, RB | 0 + RB bit | No request, RPL blocked (sent by RPL owner in idle state) |
| SF (Signal Fail) | 2 | Ring link failure detected |
| MS (Manual Switch) | 7 | Operator-initiated ring switch |
| FS (Forced Switch) | 11 | Highest priority operator-initiated switch |
| Event | - | Flush FDB notification (DNF bit controls this) |

### Ring States

| State | RPL | Description |
|-------|-----|-------------|
| Idle | Blocked | Normal operation, RPL owner blocks RPL, all other ports forwarding |
| Protection | Unblocked | Failure detected, RPL unblocked, failed link blocked |
| MS (Manual Switch) | Unblocked | Operator-triggered switch on a specific ring port |
| FS (Forced Switch) | Unblocked | Highest priority operator switch, overrides SF and MS |
| Pending | Transitioning | WTR/WTB timer running before reverting to idle |

## Basic Configuration

### IOS (Catalyst)

```
! Create ERP ring
ethernet ring g8032 RING1
 ! Define ring ports
 port0 interface GigabitEthernet1/0/1
 port1 interface GigabitEthernet1/0/2
 ! R-APS VLAN for control messages
 instance 1 aps-channel
  port0 interface GigabitEthernet1/0/1 service-instance 100
  port1 interface GigabitEthernet1/0/2 service-instance 100
  ! RPL owner configuration (only on RPL owner node)
  rpl port0 owner
  ! WTR timer in minutes (1-12, default 5)
  wtr-timer 5

! Service instance for R-APS VLAN on ring ports
interface GigabitEthernet1/0/1
 service instance 100 ethernet
  encapsulation dot1q 100
  bridge-domain 100
 !
 service instance 10 ethernet
  encapsulation dot1q 10
  bridge-domain 10

interface GigabitEthernet1/0/2
 service instance 100 ethernet
  encapsulation dot1q 100
  bridge-domain 100
 !
 service instance 10 ethernet
  encapsulation dot1q 10
  bridge-domain 10
```

### IOS-XR

```
ethernet ring g8032 RING1
 port0 interface GigabitEthernet0/0/0/0
 port1 interface GigabitEthernet0/0/0/1
 !
 instance 1
  rpl port0 owner
  ! Inclusion list — VLANs protected by this ring
  inclusion-list vlan-ids 10-100
  aps-channel
   level 3
  !
  ! Profile with timers
  profile RING-PROFILE
 !
!

! ERP profile
erp-profile RING-PROFILE
 wtr-timer 5
 guard-timer 500
 ! Hold-off timer in ms (0-10000, default 0)
 hold-off-timer 0
!
```

### NX-OS

```
! Enable G.8032 feature
feature g8032

! Create ERP instance
g8032 erp-instance RING1
 aps-channel vlan 100
 service-vlans 10-99
 rpl port Ethernet1/1 owner
 wtr-timer 5
 ring-port Ethernet1/1
 ring-port Ethernet1/2
 no shutdown
```

### JunOS

```
protocols {
    protection-group {
        ethernet-ring RING1 {
            ring-protection-link-owner;
            restore-interval 5;
            guard-interval 500;
            east-interface {
                control-channel {
                    ge-0/0/0.100;
                }
                ring-protection-link-end;
            }
            west-interface {
                control-channel {
                    ge-0/0/1.100;
                }
            }
            data-channel {
                vlan 10-99;
            }
        }
    }
}
```

## RPL Owner and Neighbor

### RPL Owner Node

```
! The RPL owner is responsible for:
! 1. Blocking the RPL in idle state
! 2. Unblocking the RPL when a failure occurs elsewhere
! 3. Sending R-APS (NR, RB) in idle state
! 4. Starting the WTR timer after failure clears

! IOS-XR RPL owner
ethernet ring g8032 RING1
 instance 1
  rpl port0 owner
```

### RPL Neighbor Node

```
! The RPL neighbor assists the RPL owner:
! 1. Also blocks the RPL in idle state (its end)
! 2. Unblocks when it receives SF from another node
! 3. Blocks again when it receives NR,RB from owner

! IOS-XR RPL neighbor
ethernet ring g8032 RING1
 instance 1
  rpl port1 neighbor
```

## Sub-Ring Topologies

### Sub-Ring with Virtual Channel

- Sub-ring connects to the major ring at two interconnection nodes
- Virtual channel provides R-APS communication across the major ring
- R-APS messages for the sub-ring traverse the major ring transparently

```
    Major Ring
  A --- B --- C --- D
  |                 |
  +----- RPL -------+

    Sub-Ring (connected at B and C)
  B --- E --- F --- C
  |                 |
  +----- RPL -------+
  (R-APS for sub-ring passes through major ring B-C)
```

```
! IOS-XR sub-ring with virtual channel
ethernet ring g8032 SUB-RING1
 port0 interface GigabitEthernet0/0/0/2
 port1 interface GigabitEthernet0/0/0/3
 open-ring
 !
 instance 1
  inclusion-list vlan-ids 200-300
  aps-channel
   level 4
  !
 !
!
```

### Sub-Ring without Virtual Channel

- R-APS messages for the sub-ring do NOT traverse the major ring
- Each interconnection node terminates the sub-ring R-APS locally
- Simpler but less flexible than virtual channel mode

## Flush FDB

### FDB Flush Mechanism

- When a topology change occurs, all nodes must flush their MAC address tables for affected VLANs
- R-APS messages carry a **DNF (Do Not Flush)** bit
- DNF=0: receiving nodes flush their FDB
- DNF=1: receiving nodes do not flush (used during sub-ring operations)
- Flush is critical for fast convergence; stale MAC entries cause traffic black-holing

## Timers

| Timer | Range | Default | Purpose |
|-------|-------|---------|---------|
| WTR (Wait to Restore) | 1-12 min | 5 min | Delay before reverting to idle after failure clears |
| Guard Timer | 10-2000 ms | 500 ms | Blocks outdated R-APS messages during state transition |
| Hold-Off Timer | 0-10000 ms | 0 ms | Delay before declaring failure (filters flaps) |

### Revertive vs Non-Revertive

```
! Revertive (default): ring returns to idle state after WTR expires
! Non-revertive: ring stays in protection state after failure clears

! IOS-XR non-revertive
ethernet ring g8032 RING1
 instance 1
  non-revertive
```

- **Revertive:** After failure clears, WTR timer starts. When WTR expires, RPL owner sends NR,RB, ring returns to idle.
- **Non-revertive:** After failure clears, ring stays in protection state. Manual operator clear required to revert.

## Multi-Ring and Ladder Topology

### Multi-Ring (Interconnected Rings)

```
    Ring 1                Ring 2
A --- B --- C         C --- D --- E
|           |         |           |
+--- RPL ---+         +--- RPL ---+

Node C is the interconnection node (member of both rings)
```

- Interconnection node participates in both rings independently
- Each ring has its own RPL, R-APS VLAN, and instance
- FDB flush in one ring can be propagated to the other via TCN (Topology Change Notification)

### Ladder Network

```
A --- B --- C --- D
|    |      |    |
E --- F --- G --- H

Two parallel rings with interconnections (ladder rungs)
```

- Modeled as a major ring with multiple sub-rings
- Each "rung" is a sub-ring connection
- Provides multiple redundancy paths

## MEF Integration

- G.8032 is commonly used in MEF (Metro Ethernet Forum) services
- E-LINE (point-to-point), E-LAN (multipoint), E-TREE services can be protected by ERP
- Service VLANs mapped to ERP instances for per-service protection
- OAM (802.1ag CFM) provides fault detection that triggers ERP switching

### Manual Switch and Forced Switch

```
! IOS-XR: Force switch on a ring port
ethernet ring g8032 RING1 instance 1 forced-switch port0

! IOS-XR: Manual switch
ethernet ring g8032 RING1 instance 1 manual-switch port0

! IOS-XR: Clear forced/manual switch
ethernet ring g8032 RING1 instance 1 clear
```

## Tips

- Only one node in the ring should be configured as RPL owner; the node at the other end of the RPL should be RPL neighbor.
- The R-APS VLAN must be dedicated to G.8032 control traffic; do not carry user data on it.
- WTR timer prevents flapping; set it long enough to avoid oscillation (5 minutes is a safe default).
- Guard timer prevents processing stale R-APS messages; 500ms is typically sufficient.
- G.8032 achieves sub-50ms protection switching, making it suitable for carrier-grade Ethernet services.
- Use CFM (802.1ag) CC (Continuity Check) messages for fast fault detection (3.3ms intervals for sub-50ms switching).
- In multi-ring topologies, ensure each ring has independent R-APS VLANs and instances to prevent interference.
- Non-revertive mode is useful when the RPL is on a low-bandwidth link and you want traffic to stay on the protection path.
- Always test failover with traffic before going to production; verify FDB flush happens correctly.
