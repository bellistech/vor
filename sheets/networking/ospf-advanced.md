# OSPF Advanced (LSA Types, Area Design, Segment Routing, and Convergence)

Advanced OSPF topics including all LSA types (1-11), area types with filtering behavior, virtual and sham links, graceful restart, segment routing extensions, BFD integration, LFA/rLFA fast reroute, OSPFv3, database filtering, and max-metric with IOS-XE and IOS-XR configuration examples.

## LSA Types Deep-Dive

### Complete LSA Type Reference

| Type | Name | Originated By | Flooding Scope | Purpose |
|:---:|:---|:---|:---|:---|
| 1 | Router LSA | Every router | Intra-area | Describes router's links and costs within area |
| 2 | Network LSA | DR on multi-access | Intra-area | Lists all routers on a broadcast segment |
| 3 | Summary LSA | ABR | Inter-area | Advertises prefixes from one area to another |
| 4 | ASBR Summary LSA | ABR | Inter-area | Advertises reachability to an ASBR |
| 5 | AS External LSA | ASBR | AS-wide | Redistributed external routes (E1/E2) |
| 6 | Group Membership LSA | MOSPF router | Intra-area | Multicast OSPF (deprecated, unused) |
| 7 | NSSA External LSA | ASBR in NSSA | NSSA area | External routes in NSSA; converted to Type 5 at ABR |
| 8 | External Attributes LSA | — | — | Reserved (RFC 2328); repurposed for OSPFv3 Link-Local |
| 9 | Opaque LSA (link-local) | Any router | Link-local | TE, graceful restart, segment routing |
| 10 | Opaque LSA (area-local) | Any router | Intra-area | TE extensions (RFC 3630), SR (RFC 8665) |
| 11 | Opaque LSA (AS-wide) | Any router | AS-wide | Inter-area TE, GRACE LSA |

### LSA Type 1 — Router LSA

```
! Every router generates exactly one Type 1 LSA per area
! Contains: router-id, link descriptions (type, id, data, metric)

! Link types within Router LSA:
!   Type 1: Point-to-point connection to another router
!   Type 2: Connection to transit network (DR present)
!   Type 3: Connection to stub network (no OSPF neighbor)
!   Type 4: Virtual link endpoint

! IOS-XE: View Router LSAs
show ip ospf database router

! IOS-XR:
show ospf database router
```

### LSA Type 2 — Network LSA

```
! Generated by the DR on multi-access (broadcast/NBMA) segments
! Contains: DR interface IP, network mask, list of attached routers

! IOS-XE:
show ip ospf database network

! Key point: No Type 2 LSA on point-to-point links (no DR election)
! Optimization: Use "ip ospf network point-to-point" on /30 or /31 links
!   to eliminate DR election and Type 2 LSAs
```

### LSA Type 3 — Summary LSA

```
! Generated by ABRs to advertise prefixes from one area into another
! ABR creates one Type 3 LSA per prefix per area boundary

! Cost = ABR's cost to reach the prefix + link cost

! IOS-XE: Inter-area route filtering
router ospf 1
 area 1 filter-list prefix BLOCK-AREA1 in    ! Filter Type 3 into area 1
 area 1 filter-list prefix BLOCK-AREA1 out   ! Filter Type 3 out of area 1

ip prefix-list BLOCK-AREA1 deny 10.99.0.0/16
ip prefix-list BLOCK-AREA1 permit 0.0.0.0/0 le 32

! IOS-XR:
router ospf 1
 area 1
  range 10.0.0.0/16                          ! Summarize Type 3 at ABR
  range 10.99.0.0/16 not-advertise           ! Suppress specific summary
```

### LSA Type 5 — AS External LSA

```
! Generated by ASBRs for redistributed routes
! Two metric types:
!   E1: External metric + internal OSPF cost to ASBR
!   E2: External metric only (default) — does not add internal cost

! IOS-XE:
router ospf 1
 redistribute bgp 65000 subnets route-map BGP-TO-OSPF metric-type 1

! Forwarding address:
!   If ASBR has OSPF-enabled interface on same subnet as the external next-hop,
!   the forwarding address is set to that next-hop IP instead of the ASBR.
!   This allows traffic to go directly to the external destination.

! IOS-XR:
router ospf 1
 redistribute bgp 65000 metric-type 1 route-policy BGP-TO-OSPF
```

### LSA Type 7 — NSSA External LSA

```
! Generated by ASBRs within NSSA areas
! Type 7 carries external routes within the NSSA
! ABR at NSSA boundary converts Type 7 → Type 5 for the rest of the AS

! IOS-XE:
router ospf 1
 area 2 nssa

! The ABR performs 7-to-5 translation:
!   - ABR with highest router-id does the translation (translator election)
!   - P-bit (Propagate): if set, ABR will translate to Type 5
!   - If P-bit clear, route stays local to NSSA

! IOS-XR:
router ospf 1
 area 2
  nssa
  nssa translate type7 always    ! Force this ABR to be translator
```

### LSA Type 9/10/11 — Opaque LSAs

```
! Opaque LSAs carry arbitrary TLV-encoded data within OSPF flooding
! Used for: TE extensions, graceful restart, segment routing, BIER

! Type 9:  Link-local scope (e.g., LLS, graceful restart)
! Type 10: Area-local scope (e.g., OSPF-TE, SR adjacency/prefix SIDs)
! Type 11: AS-wide scope (e.g., inter-area TE)

! Opaque LSA sub-types (Opaque Type field):
!   1  — Traffic Engineering LSA (RFC 3630)
!   3  — Grace LSA (graceful restart)
!   4  — Router Information LSA (capabilities, SR)
!   7  — Extended Prefix LSA (prefix SID, RFC 8665)
!   8  — Extended Link LSA (adjacency SID, RFC 8665)

! IOS-XE:
show ip ospf database opaque-area
show ip ospf database opaque-link
show ip ospf database opaque-as

! IOS-XR:
show ospf database opaque-area
```

## Area Types

### Area Type Comparison

```
                      Type 1/2   Type 3     Type 5    Type 7   Default Route
                      (Intra)    (Summary)  (External)(NSSA)   (Injected)
  ──────────────────  ─────────  ─────────  ────────  ───────  ─────────────
  Normal/Backbone     Yes        Yes        Yes       N/A      Optional
  Stub                Yes        Yes        No        No       Yes (auto)
  Totally Stubby      Yes        No*        No        No       Yes (auto)
  NSSA                Yes        Yes        No        Yes      Optional
  Totally NSSA        Yes        No*        No        Yes      Yes (auto)

  * Only the default route Type 3 LSA is allowed
```

### Stub Area Configuration

```
! IOS-XE: Stub area (blocks Type 5, ABR injects default)
router ospf 1
 area 10 stub

! Totally stubby (blocks Type 3 and Type 5)
router ospf 1
 area 10 stub no-summary              ! Only on ABR

! Stub with custom default metric
router ospf 1
 area 10 stub
 area 10 default-cost 50              ! Default Type 3 cost (default=1)

! IOS-XR:
router ospf 1
 area 10
  stub
  stub no-summary                      ! Totally stubby
  default-cost 50
```

### NSSA Configuration

```
! IOS-XE: NSSA (allows Type 7 from local ASBR, blocks Type 5)
router ospf 1
 area 20 nssa

! NSSA with default route injection
router ospf 1
 area 20 nssa default-information-originate

! Totally NSSA (blocks Type 3 and Type 5, allows Type 7)
router ospf 1
 area 20 nssa no-summary

! Suppress Type 7 to Type 5 translation
router ospf 1
 area 20 nssa no-redistribution        ! ASBR won't generate Type 7

! IOS-XR:
router ospf 1
 area 20
  nssa
  nssa no-summary                       ! Totally NSSA
  nssa default-information-originate metric 100 metric-type 1
  nssa translate type7 always           ! Force translator role
```

## Virtual Links

```
! Connect a disconnected area to the backbone (Area 0) through a transit area
! Transit area cannot be a stub/NSSA

  Area 0  ──── [ABR-1] ──── Area 1 (transit) ──── [ABR-2] ──── Area 2
                                                    ^
                             Virtual link: ABR-1 ────────────── ABR-2
                             (logically extends Area 0 through Area 1)

! IOS-XE:
! On ABR-1 (router-id 1.1.1.1):
router ospf 1
 area 1 virtual-link 2.2.2.2          ! Remote ABR's router-id
 ! With authentication:
 area 1 virtual-link 2.2.2.2 authentication message-digest
 area 1 virtual-link 2.2.2.2 message-digest-key 1 md5 SecretKey

! On ABR-2 (router-id 2.2.2.2):
router ospf 1
 area 1 virtual-link 1.1.1.1

! IOS-XR:
router ospf 1
 area 1
  virtual-link 2.2.2.2
   authentication message-digest
   message-digest-key 1 md5 SecretKey

! Verification:
show ip ospf virtual-links
```

## Sham Links (MPLS VPN)

```
! Problem: In MPLS L3VPN, two CE sites in the same OSPF domain
!   connected to different PEs. OSPF over the MPLS backbone creates
!   inter-area (Type 3) routes which are always preferred over
!   intra-area routes from the backdoor link.

  CE-1 ──── PE-1 ═══(MPLS backbone)═══ PE-2 ──── CE-2
    |                                                |
    └──────────── backdoor link (intra-area) ────────┘

! Without sham link: PE routes = inter-area (O IA) → backdoor wins
! With sham link: PE routes = intra-area (O) → MPLS backbone wins

! IOS-XE: Sham link configuration (on both PEs)
! Requires a /32 loopback in the VRF (NOT the PE loopback)
interface Loopback100
 vrf forwarding CUSTOMER-A
 ip address 10.255.0.1 255.255.255.255

router ospf 100 vrf CUSTOMER-A
 area 0 sham-link 10.255.0.1 10.255.0.2 cost 1
 ! source = local VRF loopback, dest = remote PE VRF loopback

! The VRF loopbacks must be reachable via BGP (redistribute connected in VRF)
router bgp 65000
 address-family ipv4 vrf CUSTOMER-A
  redistribute connected

! IOS-XR:
router ospf 100
 vrf CUSTOMER-A
  area 0
   sham-link 10.255.0.1 10.255.0.2
    cost 1
```

## OSPF Graceful Restart

### IETF Graceful Restart (RFC 3623)

```
! Router restarts OSPF process; neighbors act as helpers
! Forwarding plane continues using existing routes during restart

! IOS-XE: Enable graceful restart (restarting router)
router ospf 1
 nsf ietf                              ! IETF mode
 nsf ietf restart-interval 120         ! Max restart time

! IOS-XE: Enable helper mode (neighbor routers)
router ospf 1
 nsf ietf helper                       ! Act as helper (default enabled)
 nsf ietf helper disable               ! Disable helper mode
 nsf ietf helper strict-lsa-checking   ! Abort GR if topology changes

! IOS-XR:
router ospf 1
 graceful-restart
 graceful-restart interval 120
 graceful-restart helper disable       ! Disable helper (if needed)
```

### Cisco NSF (Non-Stop Forwarding)

```
! Cisco proprietary graceful restart — similar to IETF but different signaling

! IOS-XE:
router ospf 1
 nsf cisco                             ! Cisco NSF mode
 nsf cisco helper                      ! Helper mode

! NSF/SSO requires:
!   - Redundant supervisor (RSP/RP)
!   - SSO (Stateful Switchover) configured
!   - CEF maintained during switchover
```

## OSPF Segment Routing Extensions

### Prefix SID (RFC 8665)

```
! Each router advertises a Prefix SID for its loopback
! Prefix SID is an index into the SRGB (Segment Routing Global Block)

! IOS-XE:
segment-routing mpls
 global-block 16000 23999              ! SRGB range
 !
 connected-prefix-sid-map
  address-family ipv4
   10.0.0.1/32 index 1                ! SID = 16000 + 1 = 16001

router ospf 1
 segment-routing mpls
 segment-routing prefix-sid-map advertise-local

! Interface config for adjacency SIDs:
interface GigabitEthernet0/0/0
 ip ospf 1 area 0
 segment-routing mpls

! IOS-XR:
router ospf 1
 segment-routing mpls
 segment-routing forwarding mpls
 segment-routing prefix-sid-map receive disable  ! Optional
 area 0
  interface Loopback0
   prefix-sid index 1
  !
  interface GigabitEthernet0/0/0
   ! Adjacency SID assigned automatically
```

### Adjacency SID

```
! Automatically assigned per adjacency — identifies a specific link
! Used for strict (explicit) SR paths

! Types:
!   Protected adjacency SID   — eligible for FRR protection
!   Unprotected adjacency SID — no FRR

! IOS-XE verification:
show ip ospf segment-routing adjacency-sid
show ip ospf database opaque-area      ! Extended Link LSA contains adj-SID

! IOS-XR:
show ospf segment-routing adjacency-sid
```

## OSPF BFD Integration

```
! BFD provides sub-second failure detection for OSPF neighbors
! Much faster than OSPF dead interval (default 40 seconds)

! IOS-XE: Enable BFD for OSPF (per-interface)
interface GigabitEthernet0/0/0
 ip ospf bfd
 bfd interval 100 min_rx 100 multiplier 3    ! 300ms detection

! IOS-XE: Enable BFD for all OSPF interfaces
router ospf 1
 bfd all-interfaces

! Disable BFD on specific interface:
interface GigabitEthernet0/0/1
 ip ospf bfd disable

! IOS-XR:
router ospf 1
 area 0
  interface GigabitEthernet0/0/0
   bfd minimum-interval 100
   bfd multiplier 3
   bfd fast-detect
```

## OSPF Database Filter

```
! Prevent OSPF from sending LSAs out specific interfaces
! Useful for hub-and-spoke DMVPN, one-way flooding

! IOS-XE: Block outbound LSA flooding on interface
interface GigabitEthernet0/0/1
 ip ospf database-filter all out

! IOS-XR:
router ospf 1
 area 0
  interface GigabitEthernet0/0/1
   database-filter all out
```

## OSPF Max-Metric Router-LSA

```
! Advertise maximum metric (0xFFFF) to make router least preferred
! Used during maintenance, graceful insertion, or boot-up delay

! IOS-XE: Set max-metric immediately
router ospf 1
 max-metric router-lsa                          ! Indefinite max-metric
 max-metric router-lsa on-startup 300           ! 300 seconds after boot
 max-metric router-lsa on-startup wait-for-bgp  ! Until BGP converges
 max-metric router-lsa include-stub             ! Include stub links
 max-metric router-lsa external-lsa 16711680    ! Set external metric too
 max-metric router-lsa summary-lsa 16711680     ! Set summary metric

! IOS-XR:
router ospf 1
 max-metric router-lsa
 max-metric router-lsa on-startup 300
 max-metric router-lsa on-switchover 300
 max-metric router-lsa include-stub
 max-metric router-lsa external-lsa 16711680
 max-metric router-lsa summary-lsa 16711680
```

## OSPF LFA / rLFA (Loop-Free Alternates)

### IP FRR with LFA (RFC 5286)

```
! Pre-compute backup next-hop that does not loop back through the failed link
! Provides 50ms failover for single link/node failures

! LFA condition: A neighbor N is a valid LFA for destination D via S if:
!   Distance(N, D) < Distance(N, S) + Distance(S, D)
!   (N's path to D does not traverse S — the protecting router)

! IOS-XE: Enable LFA
router ospf 1
 fast-reroute per-prefix enable area 0 prefix-priority high
 fast-reroute per-prefix enable area 0 prefix-priority low
 fast-reroute per-prefix ti-lfa                ! Topology-Independent LFA

! IOS-XR:
router ospf 1
 area 0
  fast-reroute per-prefix
  fast-reroute per-prefix lfa-candidate interface GigabitEthernet0/0/0
  fast-reroute per-prefix tiebreaker node-protecting index 10
  fast-reroute per-prefix tiebreaker srlg-disjoint index 20

! Verification:
show ip ospf fast-reroute                      ! IOS-XE
show ospf fast-reroute                         ! IOS-XR
show ip route repair-paths                     ! View backup paths
```

### Remote LFA (rLFA — RFC 7490)

```
! When no direct LFA exists, tunnel to a remote repair point (PQ node)
! Uses LDP or SR tunnel to reach the repair node

  S ──── F ──── D     (S=source, F=failed link, D=destination)
  |                    No direct LFA exists
  N ──── P ──── D     P is the PQ node (repair point)
         ^
    rLFA tunnel: S ═══> P (via LDP/SR, bypassing F)

! IOS-XE: Enable rLFA
router ospf 1
 fast-reroute per-prefix enable area 0 prefix-priority high
 fast-reroute per-prefix remote-lfa tunnel mpls-ldp area 0

! IOS-XR:
router ospf 1
 area 0
  fast-reroute per-prefix remote-lfa tunnel mpls-ldp
  fast-reroute per-prefix remote-lfa maximum-cost 100  ! Limit tunnel cost
```

### TI-LFA (Topology-Independent LFA)

```
! SR-based FRR that provides 100% coverage for any topology
! Computes post-convergence path and programs SR label stack

! IOS-XE:
router ospf 1
 segment-routing mpls
 fast-reroute per-prefix enable area 0 prefix-priority high
 fast-reroute per-prefix ti-lfa

! IOS-XR:
router ospf 1
 area 0
  fast-reroute per-prefix
  fast-reroute per-prefix ti-lfa enable

! TI-LFA protection types:
!   Link protection    — protects against link failure
!   Node protection    — protects against entire node failure
!   SRLG protection   — protects against shared risk failures
```

## OSPFv3 for IPv6

```
! OSPFv3 (RFC 5340) — redesigned for IPv6 with protocol-level changes
! Key differences from OSPFv2:
!   - Runs directly over IPv6 link-local addresses
!   - Router-id and area-id remain 32-bit
!   - Authentication moved to IPsec (AH/ESP)
!   - Network addresses removed from hello/DD packets
!   - New LSA types (Link-LSA, Intra-Area-Prefix-LSA)
!   - Multiple instances per link supported

! IOS-XE:
ipv6 unicast-routing

router ospfv3 1
 router-id 1.1.1.1
 address-family ipv6 unicast
  area 0
 !

interface GigabitEthernet0/0/0
 ipv6 address 2001:db8::1/64
 ospfv3 1 ipv6 area 0

! IOS-XR:
router ospfv3 1
 router-id 1.1.1.1
 area 0
  interface GigabitEthernet0/0/0
```

### OSPFv3 Address Families (RFC 5838)

```
! OSPFv3 can carry both IPv4 and IPv6 routes using Address Families
! Single OSPFv3 process, separate instances per AF

! IOS-XE:
router ospfv3 1
 address-family ipv4 unicast
  area 0
 !
 address-family ipv6 unicast
  area 0

interface GigabitEthernet0/0/0
 ospfv3 1 ipv4 area 0
 ospfv3 1 ipv6 area 0

! IOS-XR:
router ospfv3 1
 address-family ipv4 unicast
  area 0
   interface GigabitEthernet0/0/0
 !
 address-family ipv6 unicast
  area 0
   interface GigabitEthernet0/0/0
```

## OSPF Authentication

```
! Three authentication modes: null (none), simple password, MD5/SHA

! IOS-XE: MD5 authentication (per-interface)
interface GigabitEthernet0/0/0
 ip ospf authentication message-digest
 ip ospf message-digest-key 1 md5 MySecret

! IOS-XE: SHA-256 HMAC authentication (keychain)
key chain OSPF-KEYS
 key 1
  key-string MyStrongKey
  cryptographic-algorithm hmac-sha-256

interface GigabitEthernet0/0/0
 ip ospf authentication key-chain OSPF-KEYS

! IOS-XE: Per-area authentication
router ospf 1
 area 0 authentication message-digest

! IOS-XR:
router ospf 1
 area 0
  interface GigabitEthernet0/0/0
   authentication message-digest
   message-digest-key 1 md5 MySecret

! OSPFv3 uses IPsec (AH or ESP):
! IOS-XE:
interface GigabitEthernet0/0/0
 ospfv3 authentication ipsec spi 256 sha1 <40-hex-chars>

! IOS-XR:
router ospfv3 1
 area 0
  authentication ipsec spi 256 sha1 <key>
```

## Tips

- Use totally stubby or totally NSSA areas aggressively in large deployments to minimize LSDB size; most remote sites only need a default route.
- Virtual links are a temporary fix; redesign the network to connect all areas directly to Area 0 if possible.
- Sham links require a dedicated /32 loopback in each VRF and BGP redistribution of that loopback; forgetting this is the most common sham-link failure.
- Set `max-metric router-lsa on-startup wait-for-bgp` on PE routers to prevent OSPF from attracting traffic before the BGP table is ready.
- Enable BFD on all OSPF point-to-point links for sub-second failure detection; avoid lowering OSPF hello timers below 1 second as it stresses the CPU.
- TI-LFA with SR provides 100% FRR coverage where basic LFA and rLFA cannot; deploy SR if fast reroute is critical.
- OSPFv3 address families allow running IPv4 over OSPFv3, which can simplify dual-stack deployments to a single routing protocol instance.
- Always use `auto-cost reference-bandwidth` consistently across all routers; mismatched references cause asymmetric routing.
- NSSA Type 7-to-5 translation is done by the ABR with the highest router-id; use `nssa translate type7 always` if you need deterministic translator selection.
- When using OSPF database-filter, remember it blocks ALL LSA types including Type 1; use only on spoke routers in hub-and-spoke designs.
- OSPF graceful restart helper mode is enabled by default on most platforms; disable strict LSA checking only if you understand the risk of stale forwarding.
- LFA coverage depends on topology; run `show ip ospf fast-reroute` to verify coverage percentage before relying on it in production.
